Corrige el siguiente error de tipado en el proyecto Next.js (TypeScript):

Type error: No overload matches this call.
Argument of type 'SourceInfo' is not assignable to parameter of type 'string'.


Detalles del proyecto:

Framework: Next.js 14.2.33

Lenguaje: TypeScript

Archivo con error: /src/components/VideoPlayer.tsx

Línea del error: 109

Código actual:

if (onError) plyrPlayer.on('error', (e) => onError(new Error(e.detail.plyr.source)));


Instrucciones para Gemini

Abre el archivo /src/components/VideoPlayer.tsx.

Busca la línea donde se define el callback de error de Plyr:

plyrPlayer.on('error', (e) => onError(new Error(e.detail.plyr.source)));


Corrige la llamada a new Error() para que reciba un string válido.
Reemplaza por una de estas opciones seguras:

onError(new Error(JSON.stringify(e.detail.plyr.source)));


o, si el campo src existe:

onError(new Error(e.detail?.plyr?.source?.src || 'Unknown source error'));


Asegúrate de mantener la compatibilidad de tipos con la interfaz SourceInfo.

Ejecuta npm run build para verificar que el error desaparece.

Elimina cualquier importación innecesaria y optimiza el código.
